## 1. エンジニアの「調理場」を知ろう

<Aside type="tip">

</Aside>

### 🚪 WSL(Ubuntu)にアクセスしよう
皆さんのPCには、すでにLinux（Ubuntu）がインストールされています。VS Code（エディタ）を使って、この「調理場」に入ってみましょう。

1.  **VS Codeを開く**
2.  **ショートカット `Ctrl` + `.` を押す**（または `Ctrl` + `@`）
    * 画面下に「ターミナル」という黒い窓が出てくれば成功です！
3.  **「リモート接続」の感覚**
    * これは自分のPCを操作しているようでいて、実は**「PCの中にある、もう一台の別のコンピュータ（Linux）」**にリモートでログインして操作しているような状態です。

## 2. ターミナルの基本呪文（UNIXコマンド）

エンジニアが毎日使う、基本の道具たちです。

| コマンド | 読み方 | 意味 |
| :--- | :--- | :--- |
| `pwd` | プリント・ワーキング・ディレクトリ | **現在地**を確認する |
| `ls` | リスト | 中にある**ファイルやフォルダを見る** |
| `cd [場所]` | チェンジ・ディレクトリ | 指定した**場所に移動する** |
| `mkdir [名]` | メイク・ディレクトリ | 新しい**フォルダを作る** |
| `touch [名]` | タッチ | 空の**ファイルを作る** |
| `cp [元] [先]` | コピー | ファイルを**コピーする** |
| `mv [元] [先]` | ムーブ | **移動** |
| `rm [名]` | リムーブ | **削除する**（※ゴミ箱を通らず即消えるので注意！） |
| `cat [名]` | キャット | ファイルの**中身を表示する** |
| `clear` | クリア | ターミナルの表示を**掃除する** |

## 3. 【最重要】パス（住所）の書き方

エンジニアにとって「住所」の伝え方は命です。

### 🏠 絶対パス
宇宙の根っこ（`/`）から見た完全な住所。

| OS | 住所の書き方（例） | 区切り文字 |
| :--- | :--- | :--- |
| **Windows** | `C:\Users\Name\Desktop` | `\` (バックスラッシュ) |
| **Linux / Mac** | `/home/user/projects` | `/` (スラッシュ) |

<Aside type="note">
  **なぜLinuxは `/` なの？** Windowsの `\` は日本だと「¥（円マーク）」に見えることがありますが、これは昔の名残です。インターネットやサーバーの世界基準はLinuxなので、これからは **`/` (スラッシュ)** で住所を書く癖をつけていきましょう！
</Aside>

### 📍 相対パス
「今いる場所」から見た相手の場所。
- `.` ： 今の場所
- `..` ： 一つ上の階（親フォルダ）

<Aside type="caution">
  **クイズ！** 今 `projects/app` にいます。一つ上の `projects` に戻るにはどう打つ？  
  正解は：`cd ..`
</Aside>

## 5. 【魔法】Viteで一瞬で環境を作ろう

手動でやるのは大変でしたよね？ 次は、プロが使う「**Vite（ヴィート）**」という道具を使って、モダンなWeb開発環境を一瞬で作ってみみます。

<Steps>
1.  **Viteを起動する**
    ```bash
    pnpm create vite@latest
    ```
2.  **設定を選ぶ（AIとの対話だと思って！）**
    - Project name: `my-vite-app`
    - Select a framework: `React`
    - Select a variant: `TypeScript`
3.  **フォルダに入って中身を確認**
    ```bash
    cd my-vite-app
    ls
    ```
</Steps>

<Aside type="caution">
  **見てみて！** 一瞬でたくさんのファイルができましたよね？ これが「モダン開発」のパワーです。
  手動で作った `test-app` と何が違うか、みんなで探してみよう！
</Aside>

## 6. 開発をスタートする（npm install と dev）

Viteで作った直後のファイルは、まだ「部品」が足りません。ホームセンター（npm）から道具をダウンロードして、お店をオープンさせましょう。

```bash
pnpm install  # 必要な道具（node_modules）を全部ダウンロード！
pnpm dev      # ローカルサーバーを起動！
```
### 💡 用語解説：サーバーとローカルホスト
pnpm dev を打つと http://localhost:5173 のようなURLが表示されます。

 - サーバー: データを「提供（Serve）」する役割のコンピュータ。今は君のPCが一時的にサーバーになっています。
 - localhost（ローカルホスト）: 「自分自身のコンピュータ」という意味です。
 - 自分にしか見えないので安心！: このURLは世界中に公開されているわけではなく、君のPCの中だけで動いています。ブラウザで開いて、自分の作ったものが動くか確認しましょう。

## 7. 道具を揃える「パッケージマネージャー」

アプリをインストールする方法はいくつかあります。

- **winget**: Windows用のアプリストア。
- **apt**: Linux(WSL2)用のアプリストア。
- **curl**: ネットから直接ファイルをダウンロードする魔法。

### 管理者権限 `sudo`
重要な設定を変える時は、コマンドの頭に `sudo` をつけます。
> 例：`sudo apt update`（管理者としてアプリリストを更新して！）
この際、パスワードの入力を要求されます。
パスワードは打ち込んでも表示されませんが、内部的には打ち込まれているので安心してくださいね。

## 8. VoltaとNode.jsを入れよう

Node.js（JavaScriptを動かすシェフ）を管理するために、**Volta** を使います。

<Steps>
1.  **Voltaのインストール（未導入の場合）**
    ```bash
    curl [https://get.volta.sh](https://get.volta.sh) | bash
    ```
2.  **Node.jsのインストール**
    ```bash
    volta install node@latest
    ```
3.  **バージョンの確認**
    ```bash
    node -v
    ```
    <Aside>
      `node -v` と `node --version` は同じ意味だよ！ `-v` は短く打てる便利なニックネームなんだ。
    </Aside>
</Steps>
