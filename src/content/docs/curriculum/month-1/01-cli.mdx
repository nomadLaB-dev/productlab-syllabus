## 1. 【座学】サーバーと「役割」の話

### 🖥️ サーバーの正体
「サーバー」と聞くと、冷蔵庫みたいな巨大な機械や、SF映画に出てくる光る謎の箱を想像していませんか？
実は、**サーバーの中身は、みんなが今使っているPCとほとんど同じ**です。

* **CPU:** 計算する頭脳
* **メモリ:** 作業するための机
* **HDD/SSD:** データを保管する棚

### 🎭 違いは「役割 (Role)」だけ
あなたのスマホも、「LINE」を入れれば通信機になり、「原神」を入れればゲーム機になりますよね？
コンピュータも同じで、 **「何のソフトを入れて、どう動かすか」** で呼び名が変わります。

| 役割名 | 仕事内容 | 例え |
| :--- | :--- | :--- |
| **Webサーバー** | Webページ（HTMLなど）を配る | レストランのウェイター |
| **DBサーバー** | 大事なデータを保管・整理する | 倉庫番 |

今回は、**あなたのPCの中に「仮想のサーバー環境」を作って、そこで開発を行います。**

つまり、自分のPCを実験場にするのです！

## 2. 目的に応じた使い分け & WSL (Linux) の正体

### 🐧 なぜ Windows ではなく Linux (WSL) を使うの？
多くの人のPCは Windows ですが、私たちは **WSL2 (Windows Subsystem for Linux)** という機能を使って、Windowsの中に **Linux** の環境を作ります。

**なぜわざわざそんなことをするの？**

> **答え：世界中のサーバー（本番の調理場）が Linux だからです。**

Webサービスの世界では、本番環境のほとんどが Linux で動いています。
Windows で練習していても、いざ本番環境（Linux）にコードを持っていった時に「動かない！」「コマンドが違う！」となっては大変です。
だから、**最初からプロと同じ環境（Linux）で練習する**のです。

### 🏢 じゃあ Windows はいつ使うの？
「全部 Linux でいいじゃん」と思うかもしれませんが、そうではありません。**適材適所**です。

* **Windowsが得意なこと:**
    * 📑 **事務作業:** Excel, PowerPoint, Word など
    * 🎨 **クリエイティブ:** Adobe (Photoshop, Premiere), 動画編集
    * 🎮 **エンタメ:** PCゲーム, VR
* **Linux (WSL) が得意なこと:**
    * 💻 **開発:** Webサーバー構築, プログラミング, 自動化

プロのエンジニアも、「コードを書くのはLinux (WSL)」「仕様書やチャットはWindows」といった具合に、1台のPCの中で使い分けています。

### ⚠️ 重要：OSによる「住所」の書き方の違い
Windows と Linux では、ファイルの場所（パス）を示す「区切り文字」が違います。ここが最初の罠です！

| OS | 区切り文字 | 例 |
| :--- | :--- | :--- |
| **Windows** | `\` (バックスラッシュ / ¥) | `C:\Users\Name\Desktop\` |
| **Linux / Mac** | **`/` (スラッシュ)** | `/home/user/projects/` |

<Callout type="info" emoji="💡">
  **Tips: なぜ Linux は `/` なの？**<br />
  インターネットのURLを見てみてください。 `https://google.com/search` ...全部 **スラッシュ(`/`)** ですよね？
  インターネットやWebサーバーの世界基準は **Linux (UNIX)** ベースです。
  これからは、Windowsユーザーも **住所は `/` (スラッシュ) で書く** 癖をつけましょう！
</Callout>

## 3. GUIとCUI、CLIを使いこなそう

### 🖱️ GUI vs ⌨️ CUI (CLI)
* **GUI (Graphical User Interface):** マウスでアイコンをクリックして操作する画面。直感的だけど、手順が多い。
* **CUI (Character User Interface):** キーボードで文字（コマンド）を打って命令する画面。**CLI (Command Line Interface)** とも呼びます。

エンジニアが黒い画面（CLI）を使う理由は、**「遠隔操作」と「自動化」に最強だから**です。
サーバーは世界中のどこか（北海道やアメリカ）にあります。マウスの動きをネット経由で送るとカクカクして遅いですが、文字の命令なら一瞬で届きます。

### 📍 絶対パスと相対パス
ファイル操作をする時、「どのファイルを？」を指定する方法は2つあります。

1.  **絶対パス (Absolute Path):**
    * **「住所」**で伝える方法。
    * 例: `/home/miharu/projects/app/index.js`
    * どこにいても確実に伝わりますが、長くて面倒です。
2.  **相対パス (Relative Path):**
    * **「道案内」**で伝える方法。
    * 例: `../app/index.js` （ここから一つ上の階の、appフォルダの中の...）
    * **`.` (ドット1つ):** 「今ここ（現在地）」
    * **`..` (ドット2つ):** 「一つ上の階層（親フォルダ）」

### 🛠️ 必要ソフトの点検
ターミナル（黒い画面）で以下のコマンドを打って、すべて数字（バージョン）が出れば準備完了です！

```bash
# 1. バージョン管理ツール (Volta)
volta -v
# -> 1.1.1 などの数字が出ればOK

# 2. 本体 (Node.js)
node -v
# -> v20.x.x などの数字が出ればOK

# 3. パッケージ管理ツール (npm)
npm -v
# -> 10.x.x などの数字が出ればOK
```

これらは「プログラミングをするための道具箱」です。これらが動かないと何も始まりません！

### 📜 必須コマンド（UNIX）チートシート

これだけ覚えれば、マウスなしで生きていけます。

| コマンド | 読み方 | 意味 | イメージ |
| :--- | :--- | :--- | :--- |
| **`pwd`** | プリント・ワーキング・ディレクトリ | **今どこ？** | 📍 GPSで現在地確認 |
| **`ls`** | リスト | **何がある？** | 📂 部屋の中を見渡す |
| **`cd [場所]`** | チェンジ・ディレクトリ | **移動！** | 🚶 部屋を移動する |
| **`mkdir [名前]`** | メイク・ディレクトリ | **箱を作る** | 📦 新しいフォルダ作成 |
| **`touch [名前]`** | タッチ | **紙を作る** | 📄 空のファイル作成 |
| **`cp [元] [先]`** | コピー | **複製** | 👯 コピー機 |
| **`mv [元] [先]`** | ムーブ | **引越し** | 🚚 移動、または名前変更 |
| **`rm [名前]`** | リムーブ | **消滅** | 🧨 **注意！即消えます** |
| **`cat [名前]`** | キャット | **チラ見** | 👀 ファイルの中身を表示 |
| **`clear`** | クリア | **掃除** | ✨ 画面をきれいにする |

<Callout type="warning" emoji="⚠️">
  **rmコマンドに注意！**<br />
  コマンドでファイルを消すと、ゴミ箱には入らず **完全に消滅** します。
  慣れないうちは慎重に使いましょう！
</Callout>

### ⚡️ CLI操作「3種の神器」
プロは全部タイピングしません。以下のキーを使って「サボる」のが正義です！

1. **Tabキー (自動補完):**
   コマンドやファイル名を2-3文字打って `Tab` を押すと、残りを勝手に入力してくれます。 **これを使わないと日が暮れます！**
2. **↑ キー (コマンド履歴):**
   さっき打ったコマンドを呼び出せます。「あ、スペル間違えた！」という時に便利。
3. **Ctrl + C (緊急停止):**
   「なんか変な画面から抜け出せない！」「止まらない！」という時の脱出ボタン。困ったらとりあえず押せ！

<Callout type="caution" emoji="🔑">
  **パスワード入力について**<br />
  今後、パスワードを求められる場面がありますが、**キーボードを打っても画面に `***` や `●` は出ません。**
  「壊れてる！？」と焦りますが、ちゃんと入力されています。信じてタイプしてEnterを押してください。
</Callout>

## 4. 【実践】IDEとCLIの二刀流
エンジニアは、2つの画面を行き来して開発します。

1. **IDE (VS Code)**: コードを書く場所。
2. **CLI (ターミナル)**: ファイル操作や実行を命令する場所。

### 🥊 実践演習
**Step 1**: プロジェクトを作る
ターミナルで以下のコマンドを順番に実行してください。

```bash
# 1. 自分のホームディレクトリに戻る
cd ~

# 2. プロジェクト用のフォルダを作る
mkdir my-first-project

# 3. そのフォルダの中に移動する
cd my-first-project

# 4. 今の場所を確認する
pwd
# -> /home/username/my-first-project と出ればOK！
```

**Step 2**: VS Code を起動する魔法のコマンドです。
```Bash
code .
```
※ `.` は「ここ（現在のフォルダ）」という意味でしたね。つまり **「ここをVS Codeで開け！」**

**Step 3**: コードを書いて動かす
1. VS Codeで `touch index.js` （または新規ファイル作成）
2. 中身を書く:
```JavaScript
console.log("Hello, CLI World!");
```
3. ターミナルで実行:
```Bash
node index.js
```

## 5. AI駆動開発時代の手法 / Vite
さっき、mkdir や touch でファイルを1つずつ作りましたよね？ 「正直、面倒くさい！」 と思いませんでしたか？

プロの現場では、毎回手作業でフォルダを作ることはしません。 「Scaffold（足場）」 と呼ばれる仕組みを使って、コマンド一発で最強の環境を構築します。

### ⚡ Vite (ヴィート) を体験しよう
今回は、世界中で使われている超高速ビルドツール Vite を使って、TypeScriptのプロジェクトを一瞬で作ります。

#### 1. 魔法のコマンドを唱える
ターミナルで以下を実行してください。

```Bash
npm create vite@latest
```
質問されるので、矢印キーとEnterで以下のように答えてください：

* 1. Project name: `vite-project`
* 2. Select a framework: `Vanilla` （フレームワークなし）
* 3. Select a variant: `TypeScript`

#### 2. 何が起きた？
フォルダの中を見てみましょう ( `cd vite-project` してから `ls` または `code .`)。 
さっきまで苦労して作っていたファイル群が、一瞬で生成されています！
* index.html: Webサイトの入り口
* src/main.ts: TypeScriptを書く場所
* package.json: プロジェクトの設計図

#### 3. 起動してみよう
この生成されたコードは、すぐに動かせます。

<Callout type="note" emoji="🍳">
  **npm install ってなに？**<br />
  * `package.json` = **レシピ** 「カレーを作るには肉と野菜が必要」<br />
  * `npm install` = **買い出し** 「スーパーで材料を買ってくる」<br />
  * `node_modules` = **冷蔵庫** 「材料が入っている場所」<br />
  これ（買い出し）をやらないと、料理（アプリ）が作れません！
</Callout>

```Bash
cd vite-project
npm install  # 必要な道具を一括ダウンロード
npm run dev  # 開発用サーバーを起動
```
画面に Local: http://localhost:5173/ のようなURLが出ましたか？ 
それを `Ctrl + Click` してみてください。ブラウザでWebサイトが表示されるはずです！

### 🤖 なぜこれが「AI駆動」なの？
AI（ChatGPTやGemini）にコードを書かせるとき、「ViteでVanilla TS環境で作って」 と一言伝えるだけで、AIはこのファイル構成を完璧に理解してコードを出力してくれます。 「世界標準の型（テンプレート）」を使うことが、AIを使いこなす近道なのです。
