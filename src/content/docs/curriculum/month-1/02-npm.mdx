## 1. package.json と package-lock.json

第1回で作ったプロジェクトには、2つの重要な設定ファイルがあります。

### 📄 package.json (注文書)
あなたが「これを使いたい！」と宣言するファイルです。
- `dependencies`: サービスを動かすのに必要な道具。
- `devDependencies`: 開発中（テストやビルド）にだけ使う道具。

### 🔒 pnpm-lock.yaml / package-lock.json (確定レシート)
`pnpm install` した瞬間に自動で作られる、**「実際にどのバージョンを入れたか」の全記録**です。
これがあるおかげで、チーム全員が1ビットも違わない同じ環境を再現できます。

<Aside type="danger">
  **VPoE貞末の鉄則** `pnpm-lock.yaml`（または `package-lock.json`）は、手動で編集しちゃダメ！ Gitにも必ず含めて、チーム全員で共有します。
</Aside>

## 2. バージョンの読み方：SemVer

ライブラリのバージョン番号 `1.2.3` には重要なルールがあります。



1. **メジャー (1)**：超大規模アップデート。使い方が変わるため、慎重に更新。
2. **マイナー (2)**：新機能追加。これまでの使い方はそのまま。
3. **パッチ (3)**：小さなバグ修正。

<Aside type="caution">
**🛠️ 保守運用の目安**
- **メジャー**: **「絶対にすぐ上げない」**のがプロ。ドキュメントを読み込み、移行作業の時間を確保してから挑みます。
- **マイナー**: プロジェクトに余裕がある時に上げます。新機能を使いたいならGO！
- **パッチ**: 見つけたら即上げてOK。セキュリティリスク修正が含まれることが多いです。

**目安**: 半年に1回くらい「依存関係の整理日」を作って、まとめて更新するのが実務では一般的だよ✨
</Aside>

## 3. インターネットはどう繋がっている？

ブラウザに `http://localhost:5173` と打った時、裏側で起きていること。

### 🏠 IPアドレスとDNS
- **IPアドレス**: コンピュータのインターネット上の「住所」（例: `142.250.196.110`）。
- **DNS**: 名前（google.com）を住所（IPアドレス）に変換してくれる電話帳サービス。
- **localhost**: 自分のPC自身を指す、魔法の住所（`127.0.0.1`）。



### 🚪 ポート番号（窓口）
一つの住所（PC）の中には、たくさんの窓口（ポート）があります。
- **80 / 443**: Webサイトを見るための標準の窓口。
- **5173**: Viteが開発用に一時的に開く、あなた専用の窓口。

<Aside type="tip">
**🔐 HTTPSってなに？** 昔の通信（HTTP）は「誰でも読めるハガキ」でしたが、今のWeb（HTTPS）は「本人たちしか読めない筆談」です。暗号化されているから、パスワードも安心して送れるんだよ！
</Aside>

## 4. クライアントとサーバー：どこでお料理する？

Webの世界は「おねだり（リクエスト）」と「お返し（レスポンス）」でできています。これを **HTTPプロトコル** と呼びます。

### 🍳 調理場所の違い：SPAとSSR
今のWebサイトは、お料理（HTMLの作成）をする場所が2つあります。



- **SPA (Single Page Application)**: お客さんの手元（ブラウザ）で調理。サクサク動くけど、最初は少し準備に時間がかかる。
- **SSR/SSG (Server/Static)**: お店の厨房（サーバー）で調理。完成品が届くから、表示がとっても早い！

<Aside type="tip">
3ヶ月目に学習する **Next.js** は、この「厨房」と「テーブル」を賢く使い分けられる最強の道具なんだよ！
</Aside>

## 🛡️ コラム：ソフトウェアは「生き物」である

「一度書いたコードは一生動く」と思っていませんか？ 実は、コードは放っておくと「腐敗」します。

### 1. ソフトウェアの「腐敗」
- 1年前の「当たり前」が、今は「古いやり方」になる。
- 使っているライブラリにセキュリティの穴が見つかる。
これらに対応し続けることを **「保守・運用」** と呼びます。

### 2. 具体的な保守運用の事例
- **セキュリティ対応**: 「悪い人が侵入できる穴」が見つかったら、即座にライブラリをアップデートして家（サイト）の鍵を新しくします。
- **デバイス対応**: 新しいiPhoneが出て画面が崩れたら、綺麗に見えるようにコードを微調整します。

### 3. 自動化の味方「CI/CD」
人間が毎回手作業でチェックするのは大変です。そこで「ロボット」を使います。
- **CI (継続的インテグレーション)**: コードを書き換えたら、自動で「味見（テスト）」をしてミスがないかチェックする。
- **CD (継続的デリバリー)**: チェックが通ったら、自動で「世界に公開（デプロイ）」までやってくれる。

## 🚀 今週の宿題

来週はいよいよ、自分のコードを世界に公開します！

### 1. GitHubアカウントの作成
[GitHub公式サイト](https://github.com/) で自分のアカウントを作成してください。
- ユーザー名は、エンジニアとしてずっと使う名前を考えよう。
- 二要素認証（2FA）の設定を忘れずに！

### 2. Gitの初期設定（config）
WSL2のターミナルで、自分の名前とメールアドレスを登録します。

```bash
# あなたの名前に書き換えてね（英語がおすすめ）
git config --global user.name "Your Name"

# GitHubに登録したメールアドレスに書き換えてね
git config --global user.email "your@example.com"

# 設定を確認
git config --list
```
