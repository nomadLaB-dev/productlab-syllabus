## 1. チーム開発のルール：GitHubフロー

一人の時とは違い、チームでは「みんなの共有倉庫（mainブランチ）」を壊さないことが一番大切です。

### 🤝 共同開発の基本コマンド
- **`git clone [URL]`**: 仲間が作ったプロジェクトを自分のPCにコピーする（最初だけ）。
- **`git pull origin main`**: 仲間が書いた最新のコードを自分のPCに取り込む（作業前に必ずやる！）。

### 🛠️ 誰が何をするの？（PRとレビュー）
プロの現場では、コードを直接 main には入れません。必ず **プルリクエスト（PR）** を送ります。

1.  **開発者**: ブランチを作り（`git switch -c feature/name`）、コードを書いて `push` する。GitHubで「PR」ボタンを押す！
2.  **レビュアー**: 届いたPRを読み、アドバイスをする。
    - **見るポイント**: 「動くか？」「名前が分かりやすいか？」「型が壊れていないか？」
3.  **合体（マージ）**: レビューを通ったら、mainブランチに合流！

## 3. TypeScript：なぜ「型」が必要なの？

プログラミング初心者にとって、JavaScript（JS）は自由すぎて逆に危険です。

| 特徴 | JavaScript (動的型付け) | TypeScript (静的型付け) |
| :--- | :--- | :--- |
| **例え** | 何でも入るビニール袋 | 中身が決まった専用ケース |
| **メリット** | 適当に書いても動く | **書いている最中にミスを教えてくれる** |
| **デメリット** | 動かした後に大爆発する | 最初にルール（型）を書く手間がある |

### 🚨 バグの例
```typescript
const add = (a: number, b: number) => a + b;

// JSの場合、引数が "1" (文字) でも動いてしまい、結果が "12" になる。
// TSの場合、「number（数字）じゃないよ！」と赤線を出して止めてくれる。
```

## 4. DOM操作とは？
DOM（Document Object Model） は、一言でいうとHTMLをプログラム（TypeScript）から操るための**「リモコン」**です。

### 🌳 HTMLは「ツリー（木）」でできている
ブラウザはHTMLを読み込むとき、ただのテキストとしてではなく、親子関係を持った「木の構造」として解釈します。

この「木」の枝（タグ）の一個一個を、プログラムから触れる「オブジェクト（モノ）」として扱う仕組みがDOM。

### 🕹️ リモコンで操作する流れ
DOM操作は、基本的にこの2ステップ！

1. 狙いを定める（要素を取得する）: 「あのIDのボタンを持ってきて！」
2. 命令する（変更する）: 「中身の文字を『クリックされたよ！』に変えて！」

### 💻 実際のコード例
たとえば、HTMLに 
```HTML
<div id="message">こんにちは</div> 
```
がある場合：

```TypeScript
// 1. リモコンで「message」というIDの要素を掴む
const messageElement = document.getElementById('message');

if (messageElement) {
  // 2. 命令して中身を書き換える！
  messageElement.textContent = "TypeScriptで書き換えたよ！✨";
  
  // 見た目（CSS）だって変えられちゃう
  messageElement.style.color = "blue";
}
```
<Aside type="tip" title="ワンポイント解説"> 
昔はこうやって一個ずつ「この要素を掴んで、こう変えて…」って命令（命令的UI）を書いてたんだけど、これからやる React は「このデータならこの見た目にしてね」って宣言するだけ（宣言的UI）でDOMを勝手に書き換えてくれる優れものなんだよ！ 
</Aside>

## 5. TypeScript 基本の書き方

### 💎 型を定義する（自分だけのルール）
「ユーザー」というデータの形をあらかじめ決めておけます。

```bash
type User = {
  id: number;
  name: string;
  isAdmin: boolean;
};

const member: User = {
  id: 1,
  name: "麗斗",
  isAdmin: true
};
```

## 6. JSXへの予告：HTMLを卒業しよう
これから学習する Next.js では、HTMLファイル（.html）はほとんど使いません。 代わりに、TypeScript の中で HTML っぽいものを書く「JSX」 を使います。

 - index.html → HTMLにコードを埋め込む（古いやり方）
 - page.tsx / layout.tsx → コードがHTMLを作り出す（モダンなやり方）

<Aside type="tip" title="layout.ts の役割"> 
「全ページ共通の外枠（ヘッダーやメニュー）」を担当するのが layout です。第9回で詳しくやるから、今は「共通のガワ」と覚えておいてね！ 
</Aside>
